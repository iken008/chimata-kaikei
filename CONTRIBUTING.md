# 貢献ガイド

「ちまたの会計 mini」への貢献に興味を持っていただき、ありがとうございます！

## 行動規範

このプロジェクトでは、全ての貢献者に対して敬意を持って接することを期待しています。

## 貢献方法

### バグ報告

バグを発見した場合:

1. 既存のIssueを確認し、同じ問題が報告されていないかチェック
2. 新しいIssueを作成し、以下を含めてください:
   - バグの詳細な説明
   - 再現手順
   - 期待される動作
   - 実際の動作
   - 環境情報（ブラウザ、OS等）
   - スクリーンショット（可能であれば）

### 機能提案

新機能の提案:

1. Issueを作成し、以下を含めてください:
   - 機能の詳細な説明
   - なぜこの機能が必要か
   - 可能であれば、実装のアイデア

### プルリクエスト

#### 開発環境のセットアップ

1. リポジトリをフォーク
2. ローカルにクローン:
   \`\`\`bash
   git clone https://github.com/your-username/chimata-kaikei.git
   cd chimata-kaikei
   \`\`\`

3. 依存関係をインストール:
   \`\`\`bash
   npm install
   \`\`\`

4. `.env.local` を設定 (`.env.example` を参照)

5. 開発サーバーを起動:
   \`\`\`bash
   npm run dev
   \`\`\`

#### ブランチ戦略

- `main`: 本番環境用の安定版
- `develop`: 開発用ブランチ
- 機能開発: `feature/機能名`
- バグ修正: `fix/バグ名`

#### コミットメッセージ

明確で意味のあるコミットメッセージを書いてください:

\`\`\`
feat: 新機能の追加
fix: バグ修正
docs: ドキュメントの変更
style: コードスタイルの変更（機能に影響なし）
refactor: リファクタリング
test: テストの追加・修正
chore: ビルドプロセス等の変更
\`\`\`

例:
\`\`\`
feat: 投票システムに再投票機能を追加
fix: 削除提案の集計バグを修正
docs: データベースセットアップガイドを更新
\`\`\`

#### プルリクエストのプロセス

1. 最新の `develop` ブランチから新しいブランチを作成
2. 変更を実装
3. コミット
4. フォークにプッシュ
5. プルリクエストを作成:
   - タイトル: 変更内容を簡潔に
   - 説明: 何を変更したか、なぜ変更したか
   - 関連するIssue番号を記載

#### コードレビュー

- プルリクエストは少なくとも1人のレビュアーによる承認が必要
- フィードバックには迅速かつ丁寧に対応
- 議論は建設的に

## コーディング規約

### TypeScript

- 型を明示的に定義
- `any` の使用は最小限に
- インターフェースを活用

### React/Next.js

- 関数コンポーネントを使用
- Hooksを適切に使用
- コンポーネントは単一責任の原則に従う

### スタイリング

- Tailwind CSSのユーティリティクラスを使用
- カスタムCSSは最小限に
- レスポンシブデザインを考慮

### 命名規則

- コンポーネント: PascalCase (`UserProfile.tsx`)
- 関数/変数: camelCase (`handleSubmit`, `userName`)
- 定数: UPPER_SNAKE_CASE (`DEFAULT_TIMEOUT`)
- ファイル: kebab-case (pages以外)
